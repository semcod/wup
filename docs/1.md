## Artifact Discovery, Topology, and Web Inspection

TestQL can now inspect codebases, manifests, and opt-in live URLs, then write structured topology and result artifacts for humans, CI, and LLM workflows.

```bash
# Discover local artifact types
python3 -m testql.cli discover ./testql --format json

# Build a topology graph from discovered artifacts
python3 -m testql.cli topology ./testql --format toon

# Inspect a live web page and write all data/metadata to .testql
python3 -m testql.cli inspect [https://tom.sapletta.com/](https://tom.sapletta.com/) \\
  --scan-network \\
  --out-dir .testql
```

The `.testql/` artifact bundle contains:

```text
metadata.json
topology.{json,yaml,toon.yaml}
result.{json,yaml,toon.yaml}
refactor-plan.{json,yaml,toon.yaml}
inspection.{json,yaml,toon.yaml}
summary.md
```

Live URL inspection currently extracts:

- **HTTP metadata**: status code, final URL, content type.
- **Page schema**: title, links, assets, forms.
- **Topology graph**: page, link, asset, form, interface, evidence nodes.
- **Structured results**: web checks for status, title, links, assets, forms.
- **Reports**: TOON/YAML/JSON plus NLP summary.

Example:

```bash
examples/web-inspection-dot-testql/run.sh [https://tom.sapletta.com/](https://tom.sapletta.com/)
```

Current capabilities:

- **Asset classification**: script, stylesheet, image, icon, preload, link.
- **Bounded link validation**: HEAD checks for all internal links (up to 100).
- **Bounded asset validation**: HEAD checks for all extracted assets.
- **Broken resource detection**: assets or links returning error status are flagged as findings.
- **Bounded sitemap crawl**: fetches up to 10 internal subpages, extracts titles and link counts, adds `subpage` nodes to the topology.
- **Sitemap checks**: crawl coverage, broken subpage detection, duplicate title warnings.
- **Playwright browser inspection** (`--browser`): renders the page in a headless browser, captures console errors, network calls (REST/GraphQL/WebSocket), and JS-rendered DOM.
- **Browser checks**: render detection, console error count, network call capture, title extraction, link/asset/form enumeration.

Current limitations:

- Per-resource validation uses HEAD requests only; full page content is not fetched for linked pages.
- Screenshots, performance metrics, accessibility checks, and auth flows are planned next.

## Architektura: 3-warstwowy Watcher

```
1. DETECTION LAYER (file watching + heurystyka)
   ↓ (zmiany w kluczowych folderach)
2. PRIORITY LAYER (minimalne testy powiązanych usług)
   ↓ (fail → escalate)
3. DETAIL LAYER (pełne testy z blame raportem)
```

## 1. Mapa zależności: Pliki → Endpoints → Testy

Najpierw zbuduj statyczną mapę (jednorazowo lub co 10min):

```python
# map_deps.py - generuj przy starcie
import testql.cli as tql
from pathlib import Path
import json
from collections import defaultdict

def build_dependency_map(project_root):
    # 1. Wykryj endpoints z TestQL
    endpoints = tql.endpoints(project_root, format='json')
    
    # 2. Znajdź powiązane foldery/pliki
    file_to_endpoints = defaultdict(list)
    for ep in endpoints:
        # Heurystyka: endpoint /api/users → app/users/
        module_path = ep['handler_path'].replace('app.', '').replace('.', '/')
        file_to_endpoints[module_path].append(ep['path'])
    
    # 3. Grupuj po serwisach (mikroserwisy/foldery)
    services = defaultdict(list)
    for file_path, eps in file_to_endpoints.items():
        service = '/'.join(Path(file_path).parts[:2])  # app/users → "users"
        services[service].extend(eps)
    
    return {
        'services': dict(services),
        'files': dict(file_to_endpoints)
    }

deps = build_dependency_map("./project")
json.dump(deps, open("deps.json", 'w'), indent=2)
```


## 2. Inteligentny Watcher (główny silnik)

```python
#!/usr/bin/env python3
# testql-watcher.py
import asyncio
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler
import inotify  # Linux native, 10x szybsze niż polling
from pathlib import Path
import typer
import rich
from rich.live import Live
from rich.table import Table
import subprocess
import json
import time
from collections import defaultdict, deque
import psutil  # monitor CPU/MEM

class TestQLWatcher:
    def __init__(self, project_root: str, deps_file: str = "deps.json"):
        self.project_root = Path(project_root)
        self.deps = json.loads(Path(deps_file).read_text())
        self.changed_services = set()
        self.test_queue = deque()
        self.last_test = 0
        self.cpu_throttle = 0.8  # max 80% CPU
        
    async def watch_files(self):
        # Użyj inotify (Linux) zamiast watchdog polling
        wm = inotify.WatchManager()
        mask = inotify.IN_MODIFY | inotify.IN_CREATE | inotify.IN_DELETE
        
        # Watch tylko kluczowe foldery (app/, tests/, routes/)
        paths = [str(self.project_root / "app"), 
                str(self.project_root / "tests"),
                str(self.project_root / "routes")]
        
        notifier = inotify.AsyncNotifier(wm, self.on_file_change)
        wm.watches = {p.encode(): mask for p in paths}
        
        print("🕵️  Watching:", ", ".join(paths))
        await asyncio.Event().wait()  # nieskończona pętla
    
    def on_file_change(self, iwp):
        file_path = iwp.name.decode()
        service = self.infer_service(file_path)
        
        if service and self.should_test(service):
            self.changed_services.add(service)
            self.schedule_quick_test(service)
    
    def infer_service(self, file_path: str) -> str:
        """app/users/routes.py → 'users'"""
        rel_path = Path(file_path).relative_to(self.project_root)
        parts = rel_path.parts
        if len(parts) >= 2 and parts[0] == 'app':
            return '/'.join(parts[:2])
        return None
    
    def should_test(self, service: str) -> bool:
        """Testuj tylko jeśli <5min od ostatniego testu tej usługi"""
        now = time.time()
        if hasattr(self, f'last_{service}'):
            if now - getattr(self, f'last_{service}') < 300:
                return False
        setattr(self, f'last_{service}', now)
        return True
    
    def schedule_quick_test(self, service: str):
        """Quick test: tylko endpoints tego serwisu"""
        endpoints = self.deps['services'].get(service, [])
        if endpoints:
            self.test_queue.append(('quick', service, endpoints[:3]))  # max 3 endpoints
    
    async def test_loop(self):
        while True:
            if self.test_queue and self.cpu_ok():
                test_type, service, endpoints = self.test_queue.popleft()
                
                if test_type == 'quick':
                    await self.run_quick_test(service, endpoints)
                elif test_type == 'detail':
                    await self.run_detail_test(service)
            
            await asyncio.sleep(2)  # 2s debounce
    
    async def run_quick_test(self, service: str, endpoints: list):
        cmd = [
            "testql", "scenarios/tests/smoke.testql.toon.yaml",
            "--filter-service", service,
            "--max-endpoints", "3",
            "--timeout", "10s",
            "--output", "json"
        ]
        
        result = subprocess.run(cmd, capture_output=True, text=True, timeout=15)
        data = json.loads(result.stdout)
        
        if not data.get('passed', True):
            print(f"🚨 REGRESSION in {service}!")
            self.test_queue.appendleft(('detail', service, endpoints))
    
    async def cpu_ok(self) -> bool:
        return psutil.cpu_percent(interval=0.1) < self.cpu_throttle

# CLI
app = typer.Typer()

@app.command()
def watch(project: str = ".", debounce: int = 2):
    watcher = TestQLWatcher(project)
    asyncio.run(asyncio.gather(watcher.watch_files(), watcher.test_loop()))

if __name__ == "__main__":
    app()
```
